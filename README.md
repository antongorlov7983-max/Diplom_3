# Stellar Burgers UI Tests

Автотесты для веб-приложения [Stellar Burgers](https://stellarburgers.education-services.ru/). Проверяют функциональность конструктора бургеров и ленты заказов.

## Структура проекта

- `pages/` — Page Object Model
  - `base_page.py` — базовый класс с общими методами
  - `main_page.py` — главная страница (конструктор)
  - `login_page.py` — страница авторизации
  - `feed_page.py` — лента заказов
  - `ingredient_popup.py` — всплывающие окна
- `tests/`
  - `test_constructor.py` — тесты конструктора
  - `test_order_feed.py` — тесты ленты заказов
- `conftest.py` — фикстуры (драйвер, создание пользователя)
- `data_test.py` — тестовые данные и URL

## Тестовые данные

- Пользователь создаётся через API перед тестами и удаляется после
- Уникальный email генерируется случайно
- Пароль уникальный, не вызывает предупреждений браузера

## Allure-отчёт

| Декоратор | Назначение |
|---|---|
| `@allure.feature` | Группировка по разделам (Конструктор, Лента заказов) |
| `@allure.step` | Шаги действий в отчёте |
| `@allure.title` | Название теста |

## Запуск

pip install -r requirements.txt
pytest tests --alluredir=allure-results -v
allure serve allure-results
Сценарии
### Конструктор
- Переход по клику на «Конструктор»

- Переход по клику на «Лента заказов»

- Клик на ингредиент открывает всплывающее окно

- Закрытие всплывающего окна по крестику

- Счётчик ингредиента увеличивается при добавлении в заказ

### Лента заказов
- При создании заказа счётчик «Выполнено за всё время» увеличивается

- При создании заказа счётчик «Выполнено за сегодня» увеличивается

- После оформления заказа его номер появляется в разделе «В работе»
